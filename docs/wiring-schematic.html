Reading this if colours are hard to separate: go by the line pattern first — solid-thick = +12V, solid-medium = switched, solid-black = ground, dashed = logic/sense, dotted = I²C, dash-dot = wireless. Every wire also carries a short tag (+12V / SW / GND / SIG / I2C) and a W-number that you can look up in the wire schedule, which states the type in words.

v1 control = your mechanical relay box (on/off only). The ESP drives each coil through a ULN2803A Darlington array (see detail above): the box's commoned pin 86 becomes the +12 V coil rail, each white/85 wire is low-side sunk by one channel, and the chip's COM→+12 internal clamp diodes handle flyback — so no 1N4007s on the box. FOG_EN and GRIP_EN are simple on/off enables, not PWM, and active-high (GPIO high → relay on). The external master relay gates the whole box on the engine-running logic. v2 upgrade: swap a load's relay for a logic-level MOSFET on the same enable line to gain PWM dimming — same wiring, same ESP pin, only the switching element changes.
Build order on the bike: battery − off first → mount ground bus, relay box, ESP enclosure → run + label the spine and set service loops → main fuse on battery + → build & bench-verify the ESP buck (5.0 V) before connecting the ESP → land the master-relay coil + flyback → wire each load relay (coil ← ESP enable, contact passes fused +12V to the load) → all grounds to the star → reconnect & bench-test the engine-running logic with a variable supply before fairings go back on.
Reserve for v2 (so fairings stay on): leave one spine pair unused, route an antenna path forward, and leave bay volume near the ESP for the LTE + GPS + backup-LiPo board. The relay drive, I²C bus, and ground star already accommodate it — v2 is additive, not a rewire.
